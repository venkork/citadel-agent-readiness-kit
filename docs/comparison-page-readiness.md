# Comparison page readiness

## Purpose

Help SaaS teams build **alternative and comparison pages** that agents can use for shortlisting — with fair competitor context, clear decision criteria, and no manipulative dunking.

## When to use it

- Publishing `/alternative`, `/vs`, or "compare" landing pages
- When agents misplace you against the wrong competitors
- Before listing on third-party comparison directories

## Checklist

### Structure

- [ ] Page title names **your category** and **comparison intent** ("X alternative for [audience]")
- [ ] Above-the-fold: who should choose you vs choose alternatives (decision table)
- [ ] Criteria rows are **factual and checkable** (pricing model, deployment, support, data handling)
- [ ] Primary CTA links to canonical signup or product overview — stable URL
- [ ] "When not to use us" section present

### Competitor fairness

- [ ] Named competitors are real products in the same buyer consideration set
- [ ] Claims about competitors sourced or clearly labeled as your assessment
- [ ] No humiliation, gender-war, or identity-based framing
- [ ] Updated date visible

### Agent extractability

- [ ] Comparison table in HTML/markdown, not image-only
- [ ] Pricing comparison uses **public** price points or "contact sales"
- [ ] Feature matrix distinguishes **included**, **add-on**, **not offered**

### Proof

- [ ] Sample outputs labeled illustrative where not from verified customers
- [ ] No fabricated win rates or star ratings

Link to brief template: [`../templates/comparison-page-brief.md`](../templates/comparison-page-brief.md)

## Common failure modes

- **SEO-only comparison** — thin page that agents skip for G2 or Reddit
- **Strawman competitor** — erodes trust when agents cross-check
- **Stale pricing** — damages shortlist accuracy
- **CTA to deprecated routes** — broken agent handoffs

## Safe implementation notes

- Write comparisons for **buyers**, not search engines; agents prefer decision tables over prose fluff.
- Link to your methodology or limits page when comparing qualitative features (support quality, audit depth).
- Keep a single canonical comparison URL; avoid duplicate near-copies without canonical tags.
- Refresh when competitors change pricing or positioning materially.
